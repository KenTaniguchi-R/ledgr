# SQLite to PostgreSQL Migration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace SQLite (better-sqlite3) with PostgreSQL (node-postgres) across the entire Ledgr codebase — schema, queries, actions, lib, tests, and Docker.

**Architecture:** Bottom-up migration. Schema files convert from `sqliteTable` to `pgTable`, DB driver swaps from sync better-sqlite3 to async node-postgres Pool, then all ~219 sync call sites (`.all()`, `.get()`, `.run()`) convert to async `await` patterns. Tests move from in-memory SQLite to shared Postgres testcontainer with per-file schema isolation.

**Tech Stack:** Drizzle ORM 0.45 (pg-core), node-postgres (`pg`), `@testcontainers/postgresql`, Docker Postgres 17 Alpine.

**Spec:** `docs/superpowers/specs/2026-05-10-sqlite-to-postgres-migration-design.md`

- [ ] **Step 2: Update src/lib/scoped-query.ts**

Replace `SQLiteColumn` with `PgColumn`:

```typescript
import { eq, and, type SQL } from "drizzle-orm";
import type { PgColumn } from "drizzle-orm/pg-core";
import { db as defaultDb, type LedgrDb } from "@/db";

export function scopedQuery(householdId: string, db: LedgrDb = defaultDb) {
  return {
    db,
    householdId,
    where<T extends { householdId: PgColumn }>(
      table: T,
      ...conditions: (SQL | undefined)[]
    ) {
      const filtered = conditions.filter((c): c is SQL => c !== undefined);
      return filtered.length > 0
        ? and(eq(table.householdId, householdId), ...filtered)
        : eq(table.householdId, householdId);
    },
  };
}
```

- [ ] **Step 1: Apply the conversion pattern to all query files**

For every query file, apply these mechanical transformations:

1. Every exported function that calls `.all()`, `.get()`, `.run()`, or chains off `db.select()/insert()/update()/delete()` becomes `async` and returns `Promise<...>`.
2. `.all()` → remove it, add `await` before the chain start. Drizzle's pg adapter returns arrays by default.
3. `.get()` → remove it, wrap in destructuring: `const [row] = await db.select()...limit(1)`.
4. `.run()` → remove it, add `await` before the chain start.
5. `db.transaction((tx) => { ... })` → `await db.transaction(async (tx) => { await ... })`.

Example — `getUserAiSettings` in `settings.ts`:

```typescript
// Before
export function getUserAiSettings(userId: string, db: LedgrDb = defaultDb): AiSettings | null {
  const row = db.select().from(userSettings).where(eq(userSettings.userId, userId)).get();
  // ...
}

// After
export async function getUserAiSettings(userId: string, db: LedgrDb = defaultDb): Promise<AiSettings | null> {
  const [row] = await db.select().from(userSettings).where(eq(userSettings.userId, userId)).limit(1);
  // ...
}
```

- [ ] **Step 2: Rewrite tests/integration/setup.ts**

```typescript
import { drizzle } from "drizzle-orm/node-postgres";
import { migrate } from "drizzle-orm/node-postgres/migrator";
import { Pool } from "pg";
import { randomUUID } from "crypto";
import * as schema from "../../src/db/schema";
import path from "node:path";

export async function createTestDb() {
  const connectionString =
    process.env.DATABASE_URL || "postgresql://ledgr:ledgr@localhost:5432/ledgr_test";

  const schemaName = `test_${randomUUID().replace(/-/g, "")}`;
  const pool = new Pool({ connectionString });

  await pool.query(`CREATE SCHEMA "${schemaName}"`);
  await pool.query(`SET search_path TO "${schemaName}"`);

  const db = drizzle({ client: pool, schema });
  await migrate(db, {
    migrationsFolder: path.join(process.cwd(), "src/db/migrations"),
  });

  return {
    db,
    async close() {
      await pool.query(`DROP SCHEMA "${schemaName}" CASCADE`);
      await pool.end();
    },
  };
}
```
